Introduction

This blog explains the purpose of each UML diagram required in the HBnB v2 project. We will cover the Package Diagram, the Class Diagram, the Sequence Diagrams, and finally provide a glossary of UML symbols.

Package Diagram

The package diagram organizes the application into layers (Presentation, Business Logic, Persistence). It shows dependencies between parts of the system and illustrates the Facade pattern, which simplifies interactions between layers.

Class Diagram

The class diagram describes the internal structure of the system: entities, attributes, methods, and relationships. In HBnB, our main entities are User, Place, Review, Base Model, Amenity.

Sequence Diagrams

Sequence diagrams show how the system processes an API request step by step. They illustrate the flow of messages between the client, the API, the business logic, and the database. Examples include: user registration, place creation, review submission, and fetching a list of places.
